# EICTA Data Systems Projects

This repository collects a set of academic projects developed within the *Enterprise ICT Architectures* course, covering relational, graph, and document databases, as well as basic web development.

The projects are based on a common case study: the design and implementation of data systems for managing music festivals ("Euphoric Events").

## Projects Overview

### 1. Relational Database (SQL)
**Folder:** `sql-relational-model/`

- Full database design starting from a textual scenario  
- ER modeling → logical schema → SQL implementation  
- Data population with realistic datasets  
- Advanced SQL queries (JOIN, GROUP BY, HAVING, nested queries)

**Key concepts:**
Relational modeling · normalization · constraints · analytical SQL

---

### 2. Graph Database (Neo4j)
**Folder:** `neo4j-graph-model/`

- Transformation of relational model into graph model  
- Node/relationship design and modeling choices  
- Cypher queries (create, update, delete, retrieval, aggregation)  
- Advanced queries: paths, shortest path, multi-hop analysis  

**Key concepts:**
Graph modeling · Cypher · relationship traversal · pattern matching

---

### 3. Document Database (MongoDB)
**Folder:** `mongodb-document-model/`

- Design of a single-collection document model  
- Nested sub-documents and hierarchical data  
- Implementation of CUD and retrieval queries  
- Use of aggregation pipeline and array operators  

**Key concepts:**
NoSQL · document modeling · embedding · aggregation

---

### 4. Static Web Project
**Folder:** `festival-static-website/`

- HTML/CSS website for a fictional music festival  
- Structured layout with multiple sections and pages  
- Responsive design and reusable components  

---

### 5. Web Authentication App (Node.js + SQLite)
**Folder:** `login-signup-web-app/`

- Simple login/signup system  
- Backend with Node.js + Express  
- SQLite database for user storage  
- Client-server interaction via API endpoints  

**Key concepts:**
Web development · REST APIs · authentication logic · client-server interaction

---
